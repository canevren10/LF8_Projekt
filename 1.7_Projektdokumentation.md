## 1. Projektübersicht

Ziel des Projekts war die Entwicklung einer Python-Anwendung, die Daten aus einer MariaDB-Datenbank (`heiner_it`) liest, verarbeitet und in verschiedenen Formaten ausgibt.

Das Programm bietet:
- einen **abteilungsbasierten Login** (Tkinter-GUI)
- **rollenbasierte Berechtigungen** – jede Abteilung sieht nur ihre erlaubten Tools
- verschiedene **Ausgabeformate**: Konsolenausgabe (`print`), grafische Tabelle (GUI), CSV-Export und XML-Export
- eine **eigene Erweiterung**: QR-Code-Generator für Mitarbeiterdaten (Meilenstein 2.3)

Die Anwendung wurde vollständig lokal entwickelt und getestet (macOS, MariaDB lokal).  
Der Quellcode ist in Python 3 geschrieben und nutzt ausschließlich die vorgegebenen Bibliotheken.

## 3. Probleme und Lösungen

### Problem 1 – QR-Code auf dem iPhone nicht lesbar (Meilenstein 2.3)

**Beschreibung:**  
Nach dem ersten Implementieren des QR-Code-Generators scannte das iPhone den generierten Code zwar erfolgreich, zeigte aber keinen lesbaren Text an. Stattdessen erschien die Meldung „Veri bulunamadı" (keine Daten gefunden).

**Ursache:**  
Der Text für den QR-Code wurde direkt aus dem Datenbank-Tupel erzeugt, z. B.:
```
(1, 'Clara', 'Huber', 'Sekretariat')
```
Die Klammern und Anführungszeichen machten den Inhalt für QR-Scanner-Apps unlesbar. Zusätzlich wurde das erzeugte Bild mit `LANCZOS` skaliert, was die schwarzen und weißen Felder unscharf machte.

**Lösung:**  
Die Werte werden nun per Index einzeln aus dem Tupel gelesen und als sauberer String formatiert:
```python
def qr_text(m):
    return f"Heiner IT\nName: {m[2]} {m[1]}\nAbteilung: {m[3] or '-'}"
```
Außerdem wurde die Bildgröße durch den Parameter `box_size=6` in der `QRCode`-Klasse gesteuert, ohne den QR-Code nachträglich zu skalieren. Dadurch bleibt die Bildschärfe erhalten.

---

### Problem 2 – Tkinter Combobox zeigte nicht alle Mitarbeiter (Meilenstein 2.3)

**Beschreibung:**  
Im ersten Entwurf des QR-Code-Generators wurde eine `ttk.Combobox` zur Mitarbeiterauswahl verwendet. Auf macOS öffnete sich das Dropdown-Menü der Combobox jedoch nicht vollständig – nur ein Teil der ~20 Mitarbeiter war sichtbar, der Rest war nicht erreichbar.

**Ursache:**  
Die `ttk.Combobox` hat auf macOS eine eingeschränkte Darstellung bei langen Listen. Das Dropdown-Fenster wird abgeschnitten und ist nicht scrollbar, sobald es den Bildschirmrand erreicht.

**Lösung:**  
Die Combobox wurde durch eine `tk.Listbox` mit einer `ttk.Scrollbar` ersetzt. Alle Mitarbeiter werden beim Start automatisch geladen und sind in der Liste vollständig sichtbar. Zusätzlich wurde ein Suchfeld ergänzt (wie in `06_Suche.py`), das die Liste nach Vor- oder Nachname filtert.

```python
liste = tk.Listbox(frm, height=12, width=48)
scroll = ttk.Scrollbar(frm, orient=tk.VERTICAL, command=liste.yview)
liste.configure(yscrollcommand=scroll.set)
```
